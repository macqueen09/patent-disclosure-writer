# patent-disclosure-writer

## 项目描述

用于写专利技术交底书。最简单可以只给一个业务方向即可。我用这个写专利暂时授权率是100%。

一个用于撰写、修改和评审中文发明专利技术交底书的 Codex skill。

这个 skill 总结了专利技术交底书的完整工作流：现有技术检索、创新点头脑风暴、公式与变量规范、Word 交付格式、代理机构可读的章节结构、实验支撑、以及最终自查迭代。

## 适合什么场景

- 只有一个技术想法，需要扩展成完整技术交底书。
- 已有 Word 草稿，需要按授权率导向增强。
- 需要先检索专利网站和论文，再判断创新性。
- 需要补充公式，并说明每个变量的含义和获取来源。
- 需要把公式做成 Word 里可编辑的公式格式。
- 需要给专利代理机构提供清晰、完整、可落地的技术材料。

## 使用方式

把整个 `patent-disclosure-writer` 文件夹放到支持 skill 的目录中，或发布到 GitHub 后再安装到对应的 skills 目录。

**更典型的简单使用方法：只给一个简单业务方向，也可以快速完成一篇高水平专利技术交底书。**

用户不需要一开始就准备完整算法、公式或技术方案。可以先用一句话描述业务场景，skill 会继续检索现有技术、构思可保护创新点、补充流程和模型公式，并整理成代理机构可读的技术交底书。例如：





## 核心流程

1. 读取材料  
   先读取用户提供的 Word、PDF、论文、LaTeX、实验表、模板或历史版本，确认当前最新版本。

2. 检索现有技术  
   优先检索专利网站、论文、官方文档和相邻技术，找出最接近公开方案。

3. 头脑风暴创新点  
   不只复述用户想法，还要主动寻找流程、模型、数据结构、守护机制、反馈机制、审计机制等可保护创新。

4. 组织技术交底书  
   默认章节包括背景、现有技术、缺陷、技术方案、有益效果、实验支撑、具体实施例等。

5. 公式和变量处理  
   每个公式都要有变量解释，每个变量都要说明获取来源，全文变量命名保持一致。最终 Word 中公式尽量使用可编辑公式对象。

6. 评审和修改  
   从新颖性、创造性、逻辑性、授权风险、公式完整性、Word 格式和乱码等方面做最终检查。

## 输出标准

完整任务通常应输出：

- 最终 Word 文件路径。
- 关键创新点摘要。
- 与现有技术的区别。
- 公式和变量检查结果。
- 实验支撑或实施例检查结果。
- Word 渲染或格式检查结果。
