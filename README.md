# patent-disclosure-writer

`patent-disclosure-writer` 是一个用于撰写中文发明专利技术交底书的 Codex skill。最简单只需要输入一个业务方向，它会完成现有技术检索、创新点扩展、技术方案结构化、公式与变量说明、Word 交底书生成和自检。我个人用这套流程撰写的专利，目前授权率为 100%。

## 适用场景

- 只有一个业务短语，需要快速扩展成完整专利技术交底书。
- 已有 Word 技术材料，需要继续增强创新点、公式、流程和实施例。
- 需要检索专利网站、论文、官方文档和相邻公开方案，形成差异化写法。
- 需要补充模型结构、业务闭环、数据结构、审计机制、反馈机制等可保护创新。
- 需要输出代理机构可直接阅读和继续撰写的技术交底书。
- 需要交付前检查字体、正文变量、表格变量、公式对象、乱码和非终稿语气。

## 工作流

1. 读取材料
   读取用户提供的 Word、PDF、论文、LaTeX、实验表、模板或历史版本，确认当前版本和业务目标。

2. 检索现有技术
   优先检索专利网站、论文、官方文档和相邻技术，找出最接近公开方案。

3. 构思创新点
   主动寻找流程闭环、模型公式、数据结构、守护机制、审计机制、反馈机制等可保护创新。

4. 组织技术交底书
   默认章节包括背景、现有技术、现有技术缺陷、技术方案、有益效果和具体实施例；实验数据或验证章节仅在用户要求或材料已提供时加入。

5. 处理公式和变量
   每个公式解释变量含义，每个变量说明获取来源，全文变量命名保持一致；正文、表格和实施例中的变量也同步检查，不保留 `S_j`、`τ_C`、`N_max` 这类下划线变量。

6. 最终检查
   从新颖性、创造性、逻辑性、授权风险、公式完整性、Word 格式、宋体等字体要求、乱码和非终稿语气等方面完成自检。

## 输出内容

完整任务通常输出：

- 最终 Word 文件路径。
- 关键创新点摘要。
- 与现有技术的区别。
- 授权概率和主要风险判断。
- 公式和变量检查结果。
- 章节是否符合用户要求；未要求实验数据时，不额外加入实验支撑章节。
- 终稿口径检查结果，确保没有“建议修改”“待补充”“若需要”等交付前语气。
- Word 渲染或格式检查结果。
