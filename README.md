# patent-disclosure-writer

用于写中文发明专利技术交底书的 Codex skill。最简单可以只给一个业务方向，它会继续检索现有技术、构思创新点、补充流程和公式，并整理成代理机构可读的技术交底书。我用这个流程写专利，当前阶段暂时授权率是 100%。

## 能做什么

- 从一个业务短语扩展成完整技术交底书。
- 基于 Word 草稿继续修改，按授权率导向增强。
- 检索专利网站、论文和官方资料，对比现有技术。
- 主动补充流程、模型、数据结构、审计、反馈、退役等可保护创新点。
- 增加公式，并说明每个变量的含义、获取来源和计算方式。
- 尽量把公式做成 Word 可编辑公式对象。
- 输出代理机构容易继续撰写的技术交底书结构。

## 工作流

1. 读取材料  
   读取用户提供的 Word、PDF、论文、LaTeX、实验表、模板或历史版本，确认当前最新版本。

2. 检索现有技术  
   优先检索专利网站、论文、官方文档和相邻技术，找出最接近公开方案。

3. 构思创新点  
   不只复述用户想法，还会主动寻找流程闭环、模型公式、数据结构、守护机制、审计机制、反馈机制等可保护创新。

4. 组织技术交底书  
   默认章节包括背景、现有技术、现有技术缺陷、技术方案、有益效果、实验支撑和具体实施例。

5. 处理公式和变量  
   每个公式都要解释变量含义，每个变量都要说明获取来源，全文变量命名保持一致。

6. 迭代检查  
   从新颖性、创造性、逻辑性、授权风险、公式完整性、Word 格式和乱码等方面做最终检查。

## 输出内容

完整任务通常输出：

- 最终 Word 文件路径。
- 关键创新点摘要。
- 与现有技术的区别。
- 授权概率和主要风险判断。
- 公式和变量检查结果。
- 实验支撑或实施例检查结果。
- Word 渲染或格式检查结果。
